# Getting Started

## Prerequisites

Go 1.21 or later.

```sh
go version
```

## Build

```sh
git clone <repo-url>
cd bakshell
go build -ldflags="-s -w" -o bsh ./cmd/bsh
```

Flags:

| Flag | Purpose |
|------|---------|
| `-ldflags="-s -w"` | Strips debug info, halves binary size (~3MB vs ~5MB) |
| `-o bsh` | Output binary name |

## Run

```sh
./bsh
```

On first run it creates `~/.zencr/` and `~/.zencr/plugins/` automatically. The shell works immediately with baked-in defaults — no config file required.

## Verify

```sh
echo 'echo hello world' | ./bsh
```

Expected output:

```
Welcome to BakShell!
hello world
```

## Set up config

To customize, copy the example files:

```sh
mkdir -p ~/.zencr/plugins
cp config.lua ~/.zencr/config.lua
cp plugins/*.lua ~/.zencr/plugins/
```

Restart the shell to pick up changes.

## Cross-compile for other platforms

```sh
GOOS=linux   GOARCH=amd64 go build -ldflags="-s -w" -o bsh-linux   ./cmd/bsh
GOOS=linux   GOARCH=arm64 go build -ldflags="-s -w" -o bsh-linux-arm64 ./cmd/bsh
GOOS=darwin  GOARCH=amd64 go build -ldflags="-s -w" -o bsh-darwin  ./cmd/bsh
GOOS=darwin  GOARCH=arm64 go build -ldflags="-s -w" -o bsh-darwin-arm64 ./cmd/bsh
```

## Release build

Push a `v*` tag to trigger the CI pipeline. GoReleaser cross-builds for all four targets and produces `.tar.gz`, `.deb`, `.rpm`, and `.tar.zst` packages.
